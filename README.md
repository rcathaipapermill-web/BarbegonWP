# BarbegonWP — ระบบร้านค้าโค้ดดิจิทัล/บริการ

ระบบร้านค้าเต็มรูปแบบสำหรับขายโค้ดดิจิทัล (เติมเกม, license key, gift card ฯลฯ) และบริการ
มี auth จริง (สมัคร/ล็อกอิน), ฐานข้อมูล, กระเป๋าเงินในระบบ, และระบบส่งมอบโค้ดอัตโนมัติจากสต็อก

## เทคโนโลยีที่ใช้
- **Backend**: Node.js + Express 5
- **Database**: SQLite/libSQL ผ่าน `@libsql/client` — รันเป็นไฟล์เดียวตอนพัฒนาในเครื่อง และสลับไปใช้ **Turso** (บริการ libSQL ฟรี) ตอน deploy จริงได้โดยไม่ต้องแก้โค้ด แค่เปลี่ยนตัวแปรสภาพแวดล้อม
- **Auth**: JWT เก็บใน httpOnly cookie + bcrypt สำหรับ hash รหัสผ่าน
- **View**: EJS (render ฝั่งเซิร์ฟเวอร์) + CSS ธีมมืดโทน "digital keycard"

## เริ่มต้นใช้งานในเครื่องตัวเอง (ไม่ต้องตั้งค่าอะไรเพิ่ม)

```bash
npm install
copy .env.example .env      # Windows (หรือ cp .env.example .env บน Mac/Linux)
npm start
```

เปิด http://localhost:3000 — ตอนรันครั้งแรกระบบจะสร้างไฟล์ `data/shop.db` และ seed ข้อมูลตัวอย่างให้อัตโนมัติ:
- บัญชีแอดมิน: **username `admin` / password `AdminPass123!`** — เปลี่ยนรหัสผ่านนี้ทันทีก่อนใช้งานจริง
- สินค้าตัวอย่าง 2 รายการ พร้อมสต็อกโค้ดตัวอย่าง 5 ชุด

## เอาระบบขึ้นออนไลน์แบบไม่เสียเงิน (Render + Turso)

โฮสต์ฟรีส่วนใหญ่ (รวมถึง Render แผนฟรี) **ไม่มีที่เก็บไฟล์แบบถาวร** — ถ้าใช้ไฟล์ `shop.db` ตรงๆ ข้อมูลจะหายทุกครั้งที่เซิร์ฟเวอร์รีสตาร์ทหรือ deploy ใหม่ วิธีแก้คือย้ายฐานข้อมูลไปไว้ที่ **Turso** (บริการฐานข้อมูล libSQL ที่มี free tier และเก็บข้อมูลถาวร) แล้วให้แอปต่อไปที่นั่นแทน โค้ดในโปรเจกต์นี้รองรับการสลับแบบนี้อยู่แล้ว ไม่ต้องแก้ไฟล์ใดๆ เพิ่ม

### ขั้นตอนที่ 1 — สร้างฐานข้อมูลฟรีที่ Turso

1. ไปที่ https://turso.tech แล้วสมัครบัญชี (ฟรี ไม่ต้องผูกบัตร)
2. ทำตามขั้นตอนสร้างฐานข้อมูลใหม่ 1 ฐาน (ผ่านหน้าเว็บ dashboard ของ Turso)
3. หน้า dashboard ของฐานข้อมูลจะมี 2 ค่าที่ต้องใช้:
   - **Database URL** (ขึ้นต้นด้วย `libsql://...`)
   - **Auth Token** (กดสร้าง token จากหน้าเดียวกัน)
4. เก็บ 2 ค่านี้ไว้ ใช้ในขั้นตอนถัดไป

### ขั้นตอนที่ 2 — Deploy ขึ้น Render

1. อัปโหลดโค้ดโปรเจกต์นี้ขึ้น GitHub (สร้าง repository แล้ว push โค้ดขึ้นไป)
2. ไปที่ https://render.com สมัครบัญชี (ฟรี ไม่ต้องผูกบัตร) แล้วเชื่อมต่อ GitHub
3. สร้าง **New Web Service** แล้วเลือก repository นี้ — Render จะตรวจจับว่าเป็น Node.js ให้อัตโนมัติ
4. ตั้งค่า Build Command เป็น `npm install` และ Start Command เป็น `npm start`
5. ในส่วน **Environment Variables** ของ Render ให้เพิ่ม:
   - `JWT_SECRET` = ข้อความสุ่มยาวๆ ของคุณเอง
   - `DATABASE_URL` = ค่า Database URL จาก Turso (ขั้นตอนที่ 1)
   - `DATABASE_AUTH_TOKEN` = ค่า Auth Token จาก Turso (ขั้นตอนที่ 1)
6. กด Deploy — Render จะให้ URL แบบ `https://your-app.onrender.com` มาใช้งานได้ทันที พร้อม HTTPS ให้อัตโนมัติ

จากนี้ข้อมูลร้านค้า (ผู้ใช้, ออเดอร์, สต็อกโค้ด) จะถูกเก็บที่ Turso แทนไฟล์ในเครื่อง Render ทำให้ไม่หายแม้เซิร์ฟเวอร์จะรีสตาร์ทหรือ deploy ใหม่

### ข้อจำกัดของแผนฟรีที่ควรรู้
- เซิร์ฟเวอร์ฟรีของ Render จะ "หลับ" หลังไม่มีคนเข้าใช้ 15 นาที ตอนมีคนเข้าใหม่ต้องรอ 30-60 วินาทีให้ตื่น (ถ้าอยากให้ตื่นตลอดเวลาต้องอัปเกรดเป็นแผนเสียเงิน)
- Turso free tier มีโควตาการใช้งานต่อเดือน เพียงพอสำหรับร้านค้าขนาดเล็ก-กลาง แต่ถ้ามีคนใช้เยอะมากควรตรวจสอบโควตาปัจจุบันที่หน้าเว็บ Turso

## โครงสร้างโปรเจกต์

```
server.js              จุดเริ่มต้นแอป
db/db.js                เชื่อมต่อฐานข้อมูล (local file หรือ Turso), สร้างตาราง + seed ข้อมูลเริ่มต้น
db/helpers.js           ฟังก์ชันช่วยเรียกฐานข้อมูลแบบ async (all / get / run)
middleware/auth.js      JWT verify, requireAuth, requireAdmin
routes/auth.js          สมัคร/ล็อกอิน/ล็อกเอาต์
routes/shop.js          หน้าร้าน, ซื้อสินค้า, ประวัติคำสั่งซื้อ, เติมเงิน
routes/admin.js         จัดการสินค้า/สต็อก/หมวดหมู่/ผู้ใช้/โค้ดเติมเงิน
views/                  หน้าเว็บ EJS ทั้งหมด
public/css/style.css    ธีมและดีไซน์ระบบ
```

## กติกาการทำงานของระบบ

1. **กระเป๋าเงิน**: ผู้ใช้ไม่ได้ผูกบัตรเครดิต/ชำระเงินจริง — แอดมินสร้าง "โค้ดเติมเงิน" จากหลังบ้าน (จำลองการโอนเงินแล้วให้โค้ด) แล้วผู้ใช้นำโค้ดมากรอกที่หน้า "เติมเงิน" เพื่อเพิ่มยอดในกระเป๋า
2. **สินค้าแบบโค้ดอัตโนมัติ (`code`)**: แอดมินอัปโหลดโค้ดเข้าสต็อกทีละหลายรายการ เมื่อลูกค้ากดซื้อ ระบบจะหักเงิน ล็อกโค้ด 1 รายการจากสต็อกให้ทันที และแสดงผลในหน้าคำสั่งซื้อ (การ์ดสไตล์ keycard)
3. **สินค้าแบบบริการ (`manual`)**: ระบบหักเงินและสร้างคำสั่งซื้อ แต่ไม่ส่งโค้ด — ขึ้นสถานะ "รอดำเนินการ" ให้ทีมงานติดต่อกลับเอง
4. ธุรกรรมการซื้อและการหักสต็อกอยู่ในทรานแซกชันเดียวกัน ป้องกันโค้ดถูกขายซ้ำซ้อนเมื่อมีคนซื้อพร้อมกัน

## สิ่งที่ควรทำต่อก่อนใช้งานจริง (สำคัญ)

ระบบนี้เป็นโครงสร้างที่ใช้งานได้จริงในระดับเดโม/เริ่มต้น แต่ก่อนเปิดรับผู้ใช้จริงหรือเงินจริง ควรทำเพิ่มอย่างน้อย:

- **เปลี่ยนรหัสผ่านแอดมินเริ่มต้นทันที** และตั้ง `JWT_SECRET` เป็นค่าสุ่มที่ปลอดภัย (อย่าใช้ค่า default)
- **ต่อระบบชำระเงินจริง** (เช่น เกตเวย์รับชำระของธนาคาร/ผู้ให้บริการที่มีใบอนุญาต) แทนโค้ดเติมเงินที่แอดมินสร้างเอง หากต้องรับเงินจากลูกค้าจริง การรับชำระเงินออนไลน์ในไทยมีข้อกำหนดทางกฎหมายที่ต้องปฏิบัติตาม ควรปรึกษาผู้เชี่ยวชาญด้านกฎหมาย/บัญชีก่อนเปิดรับเงินจากสาธารณะ
- เพิ่ม logging/monitoring และการสำรองฐานข้อมูลอัตโนมัติ (Turso มีระบบ backup แต่ควรตรวจสอบนโยบายปัจจุบัน)
- พิจารณาเพิ่มการยืนยันอีเมล และ rate limit เพิ่มเติมในหน้า register
- หากขยายเป็นระบบใหญ่ที่มีคนใช้พร้อมกันจำนวนมาก พิจารณาอัปเกรด Turso เป็นแผนเสียเงิน หรือย้ายไป PostgreSQL
